## 📦 技术发现

### OpenClaw 架构洞察
1. **插件系统**: 
   - 内置插件在 `/usr/local/lib/node_modules/openclaw/extensions/`
   - 用户扩展在 `~/.openclaw/extensions/`
   - 加载顺序：workspace → managed → bundled

2. **消息队列**:
   - Telegram, Discord, WhatsApp, 飞书等多渠道支持
   - 通过网关统一路由到 Agent
   - 使用 LanceDB 进行长期记忆存储

3. **安全机制**:
   - `channels.feishu.enabled: true` 表示插件已启用
   - 凭证存储在 `credentials/` 目录（需 chmod 700）

### 飞书 API 依赖
- 需要飞书开发者平台账号
- 个人开发版或企业应用两种模式
- 使用 Bot API + OAuth2.0
